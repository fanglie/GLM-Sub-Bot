```
⏳ 等待登录中... (12.0 秒)

⏳ 等待登录中... (30.2 秒)
❌ 等待登录超时（30 秒）
✅ 测试通过：超时处理正常
```

**关键发现**:

1. ✅ **登录状态检测准确**
   - 正确识别出未登录状态
   - 检测逻辑工作正常

2. ⚠️ **登录按钮未找到**
   - 可能原因：智谱 AI 的登录入口在顶部导航栏，使用了图标而非文字
   - 建议：可以手动添加导航栏登录选择器

3. ✅ **订阅按钮仍可点击**
   - 即使未登录，页面仍显示可用的"特惠订阅"按钮
   - 这说明智谱 AI 可能在点击后才检查登录状态
   - **这是一个好消息**：脚本可以直接点击，然后在支付环节前等待用户登录

4. ✅ **等待机制正常**
   - 正确等待 30 秒
   - 每 6 秒显示一次进度（第 3、6、9、12、15 次检查）
   - 超时后正确退出，没有崩溃

## 📝 改进建议

### 1. 优化登录按钮检测（可选）

当前未找到登录按钮，可以添加更多选择器：

```javascript
loginButtons: [
  'button:has-text("登录")',
  'a:has-text("登录")',
  '.login-btn',
  '[ref*="login"]',
  'text=请登录',
  // 新增：导航栏登录图标
  '.nav-login',
  '[class*="login-icon"]',
  'a[href*="/login"]',
  'a[href*="/signin"]',
]
```

### 2. 延长等待时间（可选）

如果 30 秒不够，可以修改配置：

```javascript
// src/auto-buy-glm-pro-optimized.js
const maxWaitCount = 30; // 从 15 改为 30，等待 60 秒
```

### 3. 点击后检测登录弹窗（未来优化）

如果点击按钮后弹出登录提示框，可以添加检测逻辑：

```javascript
// 点击后检查是否出现登录弹窗
if (await page.locator('.login-dialog').isVisible()) {
  logger.warning('点击后弹出登录提示，请手动登录...');
  // 等待登录完成
}
```
