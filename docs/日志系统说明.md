## 💡 使用技巧

### 1. 实时日志监控

打开**第二个终端窗口**，运行：

```bash
# Windows PowerShell
Get-Content logs\auto-buy_*.log -Tail 20 -Wait

# Linux/Mac
tail -f logs/auto-buy_*.log
```

这样可以**实时看到**脚本的每一步操作！

### 2. 导出日志为 JSON

在脚本中添加（用于详细分析）：

```javascript
// 在失败处理部分添加
logger.exportLogs('json', './failure-analysis.json');
```

然后用文本编辑器打开 `failure-analysis.json`。

### 3. 只查看错误

```bash
# Windows
Select-String -Path logs\auto-buy_*.log -Pattern "\[ERROR\]" -Context 3

# 保存到文件
Select-String -Path logs\auto-buy_*.log -Pattern "\[ERROR\]" -Context 3 > errors.txt
```

### 4. 生成时间线

```javascript
const timeline = logger.getLogs();
timeline.forEach(log => {
  console.log(`${new Date(log.timestamp).toISOString()} [${log.level}] ${log.message}`);
});
```

## ⚠️ 重要提示

### 日志文件管理

1. **自动清理**：脚本会自动保留最近 50 个日志文件
2. **手动清理**：可以安全删除 `logs/` 和 `screenshots/` 目录
3. **磁盘空间**：每次会话约 1-10MB（取决于截图数量）

### 隐私和安全

**日志可能包含**：
- 页面 URL（包含推荐链接参数）
- 网络请求头
- 元素内容

**不要公开分享日志文件**，除非已移除敏感信息！
