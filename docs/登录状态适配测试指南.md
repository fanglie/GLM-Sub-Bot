# 登录状态适配测试指南

**更新日期**: 2026-04-08  
**测试目标**: 验证脚本在登录前/登录后不同状态下的行为是否正确

---

## 📋 测试场景

### 场景 1: 未登录状态测试
**目的**: 验证脚本在用户未登录时的行为

**预期行为**:
- ✅ 检测到未登录状态
- ✅ 显示警告日志
- ✅ 等待用户手动登录（最多 30 秒）
- ✅ 登录成功后自动继续执行

**测试步骤**:
```bash
# 1. 确保已退出智谱 AI 账号
# 访问 https://www.bigmodel.cn/ 并退出登录

# 2. 运行未登录场景测试
npm run test:login-adapter:not-logged

# 或运行完整测试（包含所有场景）
npm run test:login-adapter
```

**观察点**:
1. 日志是否显示 "检测到未登录状态"
2. 是否找到登录按钮
3. 手动登录后，脚本是否自动继续
4. 登录后是否重新检查库存状态

---

### 场景 2: 已登录状态测试
**目的**: 验证脚本在用户已登录时的行为

**预期行为**:
- ✅ 检测到已登录状态
- ✅ 正常查找订阅按钮
- ✅ 不显示登录相关警告

**测试步骤**:
```bash
# 1. 确保已登录智谱 AI 账号
# 访问 https://www.bigmodel.cn/ 并登录

# 2. 运行已登录场景测试
npm run test:login-adapter:logged
```

**观察点**:
1. 日志是否显示 "检测到已登录状态"
2. 是否找到订阅按钮
3. 按钮文本和状态是否正确识别

---

### 场景 3: 按钮文本识别测试
**目的**: 验证脚本能否正确识别各种按钮文本

**预期行为**:
- ✅ 识别 Pro 连续包月按钮 (`[ref="e401"]`)
- ✅ 识别各种订阅按钮文本
- ✅ 识别售罄状态标识

**测试步骤**:
```bash
# 运行按钮识别测试（不需要特定登录状态）
npm run test:login-adapter:buttons
```

**观察点**:
1. 找到的按钮数量
2. 每个按钮的文本内容
3. 按钮是否可用（非禁用状态）
4. 是否检测到售罄标识

---

## 🔍 常见问题排查

### 问题 1: 测试失败 "Cannot find module 'playwright'"

**解决**:
```bash
npm install
npx playwright install chromium
```

---

### 问题 2: 未登录场景测试显示 "当前已登录"

**原因**: 浏览器复用了之前的登录状态

**解决**:
```bash
# 1. 手动访问 https://www.bigmodel.cn/ 并退出登录
# 2. 关闭所有 Chrome 窗口
# 3. 重新运行测试
npm run test:login-adapter:not-logged
```

---

### 问题 3: 已登录场景测试显示 "当前未登录"

**原因**: 浏览器未保存登录状态

**解决**:
```bash
# 1. 手动访问 https://www.bigmodel.cn/ 并登录
# 2. 确保登录成功（看到用户头像）
# 3. 重新运行测试
npm run test:login-adapter:logged
```

---

### 问题 4: 按钮识别测试找不到任何按钮

**可能原因**:
1. 页面尚未完全加载
2. 选择器不匹配当前页面
3. 库存已售罄，按钮被隐藏

**解决**:
```bash
# 1. 手动访问页面查看实际情况
# 访问：https://www.bigmodel.cn/glm-coding?ic=GVUM2QVEWH

# 2. 检查控制台输出的按钮列表
# 查看是否有其他文本或类名

# 3. 更新选择器配置
# 编辑 src/auto-buy-glm-pro-optimized.js 中的 SELECTORS
```

---

## 🧪 高级测试：模拟真实抢购

4. **抢购阶段** (10:00)
   - 脚本应自动继续执行
   - 观察是否成功找到并点击按钮
