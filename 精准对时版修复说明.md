# 🚨 精准对时版脚本已修复

## ✅ 修复内容

**问题**: `userDataDir option is not supported in browserType.launch`

**原因**: 精准对时版脚本错误地将 `userDataDir` 传递给了 `chromium.launch()`，但这个方法不支持该选项。

**解决方案**: 
- 如果配置了 `USER_DATA_DIR`，使用 `chromium.launchPersistentContext()` 
- 如果未配置，使用标准的 `chromium.launch()` + `browser.newContext()`

## 📋 配置说明

### 不需要保持登录状态（推荐）

编辑 `src/auto-buy-glm-pro-precision.js`：

```javascript
USER_DATA_DIR: false,  // 每次运行时手动登录
```

**优点**:
- ✅ 简单直接
- ✅ 避免用户数据目录冲突
- ✅ 适合临时使用

**缺点**:
- ❌ 每次运行需要手动登录

---

### 需要保持登录状态

编辑 `src/auto-buy-glm-pro-precision.js`：

```javascript
// Windows 示例
USER_DATA_DIR: 'C:\\Users\\你的用户名\\AppData\\Local\\Google\\Chrome\\User Data',

// Mac 示例
USER_DATA_DIR: '/Users/你的用户名/Library/Application Support/Google/Chrome/Default',
```

**注意**: 
- ⚠️ 路径中的反斜杠需要双写 `\\`
- ⚠️ 确保 Chrome 已完全关闭（包括后台进程）
- ⚠️ 第一次使用建议先手动登录一次

**优点**:
- ✅ 自动保持登录状态
- ✅ 节省登录时间

**缺点**:
- ❌ 配置复杂
- ❌ 可能与正在运行的 Chrome 冲突
